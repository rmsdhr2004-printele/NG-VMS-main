# NG-VMS Enterprise On-Premise Deployment Guide

Welcome to the **NG-VMS Sovereign Enterprise Platform**. This guide covers the installation, hardening, operations, and diagnostics of your offline, multi-tenant on-premise visitor management system.

---

## 🏗️ 1. Server Prerequisites

Ensure your virtual private server (VPS) or bare-metal host meets the following minimum specifications:

*   **Operating System:** Ubuntu 20.04 LTS / 22.04 LTS (recommended) or any modern system running Linux kernel 5.4+.
*   **Hardware Minimum:** 2 vCPUs, 4GB RAM, 20GB SSD.
*   **Hardware Recommended:** 4 vCPUs, 8GB RAM, 50GB NVMe SSD (highly recommended for high-volume face recognition matching).
*   **Network Firewall Bounds:** Only ports **`80` (HTTP)** and **`443` (HTTPS)** need public inbound accessibility. All internal microservices (MongoDB, Redis, MinIO, Alerting, Grafana) are strictly bound to internal interfaces and insulated from external connections.

---

## 🚀 2. Zero-Touch Installation Flow

Our automated orchestrator handles container runtimes, database dependencies, host directory permissions, network firewalls, and dynamic security secrets automatically.

### Step 1: Unpack the Deliverable
Connect to your host machine via SSH, create your target directory, and extract the enterprise bundle:
```bash
# Create deploy directory
mkdir -p /opt/ngvms && cd /opt/ngvms

# Extract tar package (Assuming bundle is delivered to /tmp)
tar -xzf /tmp/ngvms-enterprise.tar.gz -C .
```

### Step 2: License Key Placement
Place your enterprise license signature file `license_NGS.lic` in the root of the deployment directory:
```bash
# Verify license is placed correctly
ls -la license_NGS.lic
```

### Step 3: Environmental Setup
Copy the production environment template and lock down the configuration file's permissions:
```bash
# Copy template
cp .env.example .env

# Lock permissions (Allows only the owner to read/write system secrets)
chmod 600 .env
```
*(Optional) You can open `.env` using your text editor (`nano .env`) to set custom host domain name routing (`DOMAIN_NAME`) and corporate SMTP mailets. All cryptographic keys are automatically generated by the installer if left blank!*

### Step 4: Run the Installer
Trigger the gold-standard installer. The script will automatically verify dependencies, lock UFW firewall rules, generate secure random secrets, seed Mongo replica set keys, start the containers, and run health diagnostics:
```bash
# Trigger installer (Requires root/sudo privileges)
sudo ./install.sh
```

## 🩺 3. Diagnostics & Health Monitoring

To verify system responsiveness, compute resources, database replication synchronization, and secure caching layers at any time, run our diagnostic engine:
```bash
# Query active health statuses
sudo ./healthcheck.sh
```

### Response Codes:
*   `OK` (Exit Code 0): Everything is fully functional and synchronized.
*   `CRITICAL` (Exit Code 1): An active problem was detected (e.g. database down, storage exceeded). Fault details will be printed to stdout.

---

## 📦 4. Persistent Storage Backups & Disaster Recovery

Disaster Recovery and system updates are fully automated to guarantee continuous operation.

### Running a Backup
Creates a compressed, self-contained database and asset snapshot in the `/backups` directory:
```bash
# Trigger automated backup
sudo ./backup.sh
```

### Restoring from a Backup
To restore the platform to a previous snapshot state, supply the backup file as an argument:
```bash
# Restore to a specific archive (Warning: Destructive overwrite)
sudo ./restore.sh ./backups/ngvms_backup_2026-05-18_12-00-00.tar.gz
```

### Upgrading the System
To apply hot-updates, simply run our safe updater. It automatically snapshots your existing database state before pulling container revisions:
```bash
# Perform safe update
sudo ./update.sh
```
